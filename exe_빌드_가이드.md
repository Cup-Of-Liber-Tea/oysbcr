# 올리브영 리뷰 수집기 EXE 빌드 가이드

## 빌드 방법

### 1. 자동 빌드 (권장)
```bash
python build_exe.py
```
또는
```bash
build.bat
```

### 2. 빌드 옵션
- **폴더 형태**: 안정성이 높지만 여러 파일로 구성
- **단일 exe**: 배포가 편리하지만 크기가 크고 느릴 수 있음

## 캡차 문제 해결

### 문제점
exe 파일에서 셀레니움베이스가 제대로 작동하지 않을 수 있습니다.

### 해결방법

#### 1. 브라우저 드라이버 수동 설치
```bash
# 셀레니움베이스 설치 후 브라우저 설치
pip install seleniumbase
seleniumbase install chrome
```

#### 2. 환경변수 설정
Windows에서 다음 환경변수를 설정하세요:
- `PATH`에 Chrome 설치 경로 추가
- `CHROMEDRIVER_PATH` 설정 (선택사항)

#### 3. exe 실행 전 준비사항
exe 파일을 실행하기 전에:
1. Chrome 브라우저가 설치되어 있는지 확인
2. 인터넷 연결 확인
3. 방화벽/백신 프로그램에서 exe 파일 허용

## 빌드 시 포함되는 라이브러리

### 기본 라이브러리
- pandas: 데이터 처리
- openpyxl: 엑셀 파일 생성
- requests: HTTP 요청
- urllib3: SSL 처리

### 셀레니움 관련
- seleniumbase: 브라우저 자동화
- selenium: 웹드라이버
- 브라우저 드라이버 (Chrome)

### GUI 관련
- tkinter: 사용자 인터페이스
- threading: 멀티스레딩

## 문제 해결

### 1. "모듈을 찾을 수 없습니다" 오류
```bash
# 누락된 라이브러리를 hidden-import에 추가
--hidden-import=누락된모듈명
```

### 2. 셀레니움 관련 오류
- Chrome 브라우저 최신 버전 설치
- 셀레니움베이스 재설치: `pip install --upgrade seleniumbase`

### 3. SSL 인증서 오류
- 인터넷 연결 확인
- 방화벽 설정 확인
- 회사 네트워크의 경우 프록시 설정 확인

### 4. 권한 오류
- 관리자 권한으로 실행
- 백신 프로그램에서 exe 파일 예외 처리

## 배포 시 주의사항

### 폴더 형태 배포
- 전체 폴더를 압축하여 배포
- exe 파일만 따로 복사하면 실행되지 않음

### 단일 exe 배포
- 파일 크기가 클 수 있음 (100MB 이상)
- 첫 실행 시 압축 해제로 인해 느릴 수 있음

## 성능 최적화

### 빌드 크기 줄이기
```bash
# UPX로 압축 (선택사항)
pip install upx
pyinstaller --upx-dir=upx경로 ...
```

### 실행 속도 향상
- 폴더 형태 빌드 사용
- SSD에 설치
- 백신 실시간 검사에서 제외

## 테스트 방법

1. 빌드 완료 후 exe 파일 실행
2. 상품 ID 입력 (예: A000000200614)
3. 캡차 화면이 나오는지 확인
4. 캡차 해결 후 리뷰 수집이 시작되는지 확인
5. 엑셀, JSON 파일이 정상 생성되는지 확인
